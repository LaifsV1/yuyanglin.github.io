Software Foundations & Verification Group
School of Computer Science and Statistics, Trinity College Dublin

I am currently a Research Fellow at Trinity College Dublin. My main interest is in formal methods, verification and programming languages. In particular, I am interested in the development of tools that automatically reason about programs, especially those written in higher-order languages.

Research Tools

  • YulTracer (2024)
    Initial Interpreter [Alpha 0.1.1] (DOI: 10.5281/zenodo.12098663)
    Tool Artefact [Alpha 0.1.1] (DOI: 10.5281/zenodo.12588075)
    Modular interpreter for Yul and future symbolic execution tool for Yul. Based on our own operational semantics for Yul. The interpreter is designed to be modular on so-called "dialects" and comes with a partial implementation of the EVM Execution Specification (Shanghai upgrade) as its only dialect. Developed in OCaml with Z3 bindings.
    Associated report for Yul semantics [Alpha 0.1.1] (DOI: 10.48550/arXiv.2407.01365)
  • Hobbit-PDNF (2024)
    Initial Release (DOI: 10.5281/zenodo.11128049)
    Equivalence verification tool for open higher-order programs with local state based on Symbolic Execution, Pushdown Normal-Form Bisimulation which leverages the decidability of reachability in pushdown systems, and Bisimulation Up-To Techniques. Developed in OCaml with Z3 bindings.
    Associated paper (DOI: 10.1145/3661814.3662103)
  • pcfeq (2023)
    Initial Release (DOI: 10.5281/zenodo.7864539)
    Equivalence verification tool for call-by-value PCF based on Symbolic Execution, a Normal-Form Bisimulation for functional languages based on Nominal Game Semantics, and Bisimulation Up-To Techniques. Developed in OCaml with Z3 bindings.
    Associated paper (DOI: 10.1109/LICS56636.2023.10175778)
  • Hobbit (2021)
    Initial Release (DOI: 10.5281/zenodo.7864658)
    Equivalence verification tool for open higher-order programs with local state based on Symbolic Execution, a Normal-Form Bisimulation derived by combining Nominal Game Semantics with Environmental Bisimulation, and Bisimulation Up-To Techniques. Developed in OCaml with Z3 bindings.
    Associated paper (DOI: 10.1007/978-3-030-99527-0_10)
  • HOLiK (2019)
    Safety verification tool for open higher-order programs with global state based on Symbolic Execution and Nominal Game Semantics. Developed in the K Framework.
    Associated paper (DOI: 10.4230/LIPIcs.FSCD.2020.27)
  • BMC2 (2018)
    Safety verification tool for closed higher-order programs with global state based on Bounded Model Checking via SMT solving and Nominal Defunctionalisation. Developed in OCaml with Z3 bindings.
    Associated paper (DOI: 10.1007/978-3-030-35540-1_1)
  • YUP (2015)
    Proof checker for simple program-correctness proofs of functional programs through inductive and equational reasoning based on sequent calculus and abstract congruence closure. YUP was designed with the aim of teaching about proof formalisation. Developed in OCaml.

Personal Projects

  • 2x2x3-Simple-Simulator (2022)
    Simple CLI simulator for a 2x2x3 twisty puzzle. Written to find a generic solution for the 2x2x3 cube. Uses BFS/DFS and explore-set memoisation to search for a path from one configuration to another. Found algorithm to swap two adjascent pieces: r f t'r t f t'f t r t'. Written in OCaml.
  • CountdownSolver (2022)
    Solver for a puzzle game similar to the Numbers round from the Countdown game show (i.e. finds all ways to arithmetically produce a given number from a given set of factors). Based on search with normalisation and memoisation, and rules to prune symmetry and identity. Written in OCaml.
  • C4 (2019)
    Interpreter for a 4-counter-machine language based on the BF esoteric language. Written in OCaml.
